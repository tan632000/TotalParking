# Modular Workflow: Brainstorm

**Slash Command:** `/brainstorm`

This workflow helps you generate ideas, define user personas, and clarify the core value of a feature.

## Steps

1.  **Context & Pain Points**:
    -   Interview User: "What problem are we solving?"
    -   Identify current bottlenecks or gaps.

2.  **Solution Generation**:
    -   Brainstorm potential solutions.
    -   Assess Pros/Cons of each solution.

3.  **User Personas**:
    -   Define who is this for? (e.g., "Creator", "Admin", "End-user").
    -   Draft User Personas in `docs/{{feature}}/ideas.md`.

4.  **Output**:
    -   Create or update `docs/{{feature}}/ideas.md` with:
        -   Problem Statement
        -   Proposed Solutions
        -   Selected Approach
        -   Key User Personas
