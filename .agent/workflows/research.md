# Modular Workflow: Research

**Slash Command:** `/research`

This workflow focuses on technical feasibility, design intelligence, and finding the right tools for the job.

## Steps

1.  **Tech Research**:
    -   Search for libraries, APIs, or existing solutions.
    -   Evaluate compatibility with current stack (Next.js/FastAPI).
    -   Draft technical options in `docs/{{feature}}/research.md`.

2.  **Design Intelligence**:
    -   Use `UI UX Pro Max` skill (if installed) or browse best practices.
    -   Identify UI patterns, color schemes, or interactions.
    -   Save findings to `docs/{{feature}}/research.md`.

3.  **Competitor Analysis** (Optional):
    -   Review how others solve this problem.
    -   Note down key takeaways.

4.  **Output**:
    -   Update `docs/{{feature}}/research.md` with:
        -   Tech Stack choices (Libraries, Tools).
        -   Design Inspiration (Links, Descriptions).
        -   Feasibility Conclusion (Go/No-Go).
