# Fast Track Workflow

**Slash Command:** `/quick-change` (or `/fix`)

This workflow is for small tasks, bug fixes, and minor tweaks that do not require a full proposal process.

## Process

1.  **Context**:
    -   Read relevant files.
    -   Understand the specific requirement or bug report.

2.  **Implementation**:
    -   Make code changes directly.
    -   **Rule**: Keep changes minimal and focused.

3.  **Verification**:
    -   Run existing tests to ensure no regressions.
    -   Review usage of `render_diffs` or `diff` to verify changes.

4.  **Finish**:
    -   Commit changes with a descriptive message.
